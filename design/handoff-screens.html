EditSetSheet
Inline component — ActiveWorkoutScreen
Bottom sheetPR warning on open
Trigger
Long-press or swipe-left on any completed set row in the set table. Shows sheet with that set pre-filled.
PR warning rule
Show immediately on sheet appear if the set being edited is the user's current PR for this exercise. No delay. Uses .onAppear, not a save-triggered check.
Layout — bottom sheet, detent 62% screen height
Implementation: .sheet(isPresented:) with .presentationDetents([.fraction(0.62)]) · .presentationDragIndicator(.visible) · .presentationBackground(Color.prBackgroundSec)
Handle: system drag indicator (not custom)
Header: "Edit Set N" prHeadlineLarge · "Cancel" 14pt prBrandLight button right · padding 18pt
PR warning banner: prWarning 10% bg · prWarning 30% border · warning triangle + "Editing this set will recalculate your personal record" · 13pt · shown on .onAppear
Inputs HStack gap 10pt:
— Weight field: prBackgroundTer bg · height 50pt · prDataMedium monospaced · focused: 1.5pt prBrandLight border + outer glow
— Reps field: same
Save button: "Save changes" PRButton(.primary) height 48pt · margin 16pt horizontal
Divider: prBorder · margin 16pt horizontal
Delete set: "Delete set" 15pt bold prError · centred · taps → confirmation alert (not another sheet)
Delete set confirmation: use SwiftUI .alert("Delete set?", isPresented:) with destructive button "Delete" and cancel. Never use a sheet-within-a-sheet.
11
Delete Workout — Confirmation Dialog
Inline component — WorkoutSummaryScreen + WorkoutHistoryScreen
DestructiveTwo surfaces
Surfaces
  • WorkoutSummaryScreen — "Delete workout" option in ⋯ menu or swipe action
  • WorkoutHistoryScreen — swipe-left on workout row → delete action
Post-deletion
  • Dismiss dialog
  • Pop navigation to HomeScreen (if from Summary) or refresh list (if from History)
  • PR recalculation runs in Core Library asynchronously
Implementation
Preferred: SwiftUI .confirmationDialog() — renders as system action sheet on iOS, matches platform HIG.
Custom centered dialog (as shown in spec): only if product requires custom styling. Use ZStack overlay with .background(.ultraThinMaterial) scrim. 
// Option A — native (preferred)
.confirmationDialog(
    "Delete workout?",
    isPresented: $showDeleteConfirm,
    titleVisibility: .visible
) {
    Button("Delete workout", role: .destructive) { viewModel.deleteWorkout() }
    Button("Cancel", role: .cancel) {}
} message: {
    Text("This will permanently delete this workout and all its sets. Your personal records may be updated.")
}

// Option B — custom (if design requires exact spec visual)
if showDeleteConfirm {
    ZStack {
        Color.black.opacity(0.58).ignoresSafeArea()
            .onTapGesture { showDeleteConfirm = false }
        PRDeleteDialogCard(
            onDelete: { viewModel.deleteWorkout(); showDeleteConfirm = false },
            onCancel: { showDeleteConfirm = false }
        )
    }
    .transition(.opacity.animation(PRAnimation.standard))
}
Never merge PRs without deletion completing successfully. The deleteWorkout() call must confirm backend deletion before dismissing. Show a loading state on the "Delete workout" button if the delete call takes >0.5s.
